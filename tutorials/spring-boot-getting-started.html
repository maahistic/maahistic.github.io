1. Introduction

Spring Boot 3 is a major milestone for the Spring ecosystem. It brings Java 17+ as the minimum baseline, migrates from Java EE to Jakarta EE 10, adds native image support via GraalVM, and improves observability with Micrometer.

In this tutorial, we'll build a clean Task Manager REST API from scratch — with proper layering (Controller → Service → Model) following Spring Boot best practices.

By the end, you'll have a working API that you can extend with a database, authentication, and more.

2. Prerequisites

  • Java 17+ installed — run java --version to verify
  • Maven 3.8+ or Gradle 7.5+
  • An IDE — IntelliJ IDEA (recommended), VS Code, or Eclipse
  • Basic Java knowledge (classes, interfaces, annotations)
Tip

If you're on Windows, use winget install EclipseAdoptium.Temurin.17.JDK to install Java 17 quickly.

3. Creating the Project

The easiest way is Spring Initializr. Configure it as follows:

  • Project: Maven
  • Language: Java
  • Spring Boot: 3.2.x (latest stable)
  • Group: com.example
  • Artifact: task-api
  • Dependencies: Spring Web

Click Generate, unzip the download, and open it in your IDE.

Alternatively, use the command line:

Terminalcurl https://start.spring.io/starter.zip \
  -d dependencies=web \
  -d type=maven-project \
  -d language=java \
  -d javaVersion=17 \
  -d groupId=com.example \
  -d artifactId=task-api \
  -o task-api.zip

unzip task-api.zip -d task-api
cd task-api

The key file is TaskApiApplication.java — the entry point:

TaskApiApplication.java@SpringBootApplication
public class TaskApiApplication {
    public static void main(String[] args) {
        SpringApplication.run(TaskApiApplication.class, args);
    }
}

The @SpringBootApplication annotation combines three things: @Configuration, @EnableAutoConfiguration, and @ComponentScan. Spring Boot handles the rest.

5. Your First REST Endpoint

Let's create a simple health check endpoint. Create a new file HelloController.java:

HelloController.javapackage com.example.taskapi.controller;

import org.springframework.web.bind.annotation.*;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello from Spring Boot 3! 🚀";
    }
}

Run the application:

Terminal./mvnw spring-boot:run

Open http://localhost:8080/hello in your browser — you'll see the greeting.

Note

@RestController = @Controller + @ResponseBody. Every method return value is serialized directly as the HTTP response (typically JSON).

6. Adding Model & Service Layer

Real APIs need structure. Let's build a proper Task Manager with three layers.

The Model — Task Record

Java 17 records are perfect for immutable data objects:

model/Task.javapackage com.example.taskapi.model;

public record Task(Long id, String title, boolean completed) {}

The Service — Business Logic

service/TaskService.javapackage com.example.taskapi.service;

import com.example.taskapi.model.Task;
import org.springframework.stereotype.Service;
import java.util.*;
import java.util.concurrent.atomic.AtomicLong;

@Service
public class TaskService {

    private final List<Task> tasks = new ArrayList<>();
    private final AtomicLong counter = new AtomicLong(1);

    public List<Task> findAll() {
        return Collections.unmodifiableList(tasks);
    }

    public Task create(String title) {
        Task task = new Task(counter.getAndIncrement(), title, false);
        tasks.add(task);
        return task;
    }

    public Optional<Task> toggleComplete(Long id) {
        for (int i = 0; i < tasks.size(); i++) {
            if (tasks.get(i).id().equals(id)) {
                Task old = tasks.get(i);
                Task updated = new Task(old.id(), old.title(), !old.completed());
                tasks.set(i, updated);
                return Optional.of(updated);
            }
        }
        return Optional.empty();
    }
}

The Controller — REST Endpoints

controller/TaskController.javapackage com.example.taskapi.controller;

import com.example.taskapi.model.Task;
import com.example.taskapi.service.TaskService;
import org.springframework.web.bind.annotation.*;
import java.util.*;

@RestController
@RequestMapping("/api/tasks")
public class TaskController {

    private final TaskService taskService;

    // Constructor injection (recommended over @Autowired)
    public TaskController(TaskService taskService) {
        this.taskService = taskService;
    }

    @GetMapping
    public List<Task> getAllTasks() {
        return taskService.findAll();
    }

    @PostMapping
    public Task createTask(@RequestBody Map<String, String> body) {
        return taskService.create(body.get("title"));
    }

    @PatchMapping("/{id}/toggle")
    public Task toggleTask(@PathVariable Long id) {
        return taskService.toggleComplete(id)
            .orElseThrow(() -> new RuntimeException("Task not found"));
    }
}

7. Testing Your API

Start the application and test with cURL:

Terminal# Create a task
curl -X POST http://localhost:8080/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "Learn Spring Boot 3"}'

# Response: {"id":1,"title":"Learn Spring Boot 3","completed":false}

# Get all tasks
curl http://localhost:8080/api/tasks

# Toggle completion
curl -X PATCH http://localhost:8080/api/tasks/1/toggle

# Response: {"id":1,"title":"Learn Spring Boot 3","completed":true}
Tip

Use Postman or Swagger UI (add springdoc-openapi-starter-webmvc-ui dependency) for a visual API testing experience.

8. Next Steps

You now have a working REST API! Here's where to go from here:

  • Add a database — Use Spring Data JPA with H2 or PostgreSQL
  • Add validation — Use @Valid and Bean Validation annotations
  • Error handling — Create a @ControllerAdvice for consistent error responses
  • Containerize it — Check out my Docker for Java Developers tutorial
  • Add tests — Use @WebMvcTest and MockMvc for controller tests
Spring Boot's magic lies in convention over configuration — it gives you sensible defaults and lets you override only what you need.
